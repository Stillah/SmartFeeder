# SmartFeeder API

Backend API для умной кормушки (проект SmartFeeder). Обеспечивает управление питомцами, расписанием кормлений, сбор статистики (история и анализ) и работу с изображениями с камеры (включая сохранение эмбеддингов для классификации).

## Технологический стек
- **Фреймворк:** FastAPI
- **База данных:** PostgreSQL + расширение `pgvector` (для хранения и поиска по эмбеддингам)
- **ORM:** SQLAlchemy (асинхронная)
- **Валидация:** Pydantic
- **Контейнеризация:** Docker & Docker Compose
- **Тестирование:** pytest, httpx

## Быстрый старт

### 1. Настройка переменных окружения
В корне проекта находится файл `.env.example`. Скопируйте его в `.env`:
```bash
cp .env.example .env
```
При необходимости отредактируйте `.env` (по умолчанию настройки подходят для локального запуска через Docker).

### 2. Запуск через Docker Compose
Поднимите контейнеры (базу данных и само приложение) с помощью команды:
```bash
docker-compose up --build
```
*Для запуска в фоновом режиме добавьте флаг `-d`.*

После успешного запуска API будет доступно по адресу: **http://localhost:8000**

### 3. Документация API (Swagger)
Интерактивная документация и возможность протестировать все эндпоинты доступны по адресу:
👉 **http://localhost:8000/docs**

## Тестовые данные (Seeding)
При **первом запуске** на пустой базе данных скрипт автоматически создаст тестовые данные для удобства проверки:
- Тестового пользователя и питомца.
- Расписание кормления.
- Статус кормушки.
- Логи кормлений (для проверки `/history/{pet_id}/recent` и `/history/{pet_id}/analysis`).
- 3 тестовых изображения (прозрачные GIF 1x1) с сгенерированными эмбеддингами (сохраняются в `storage/images/`).

**После запуска**:
```bash
docker-compose down
```
*(Внимание: флаг `-v` удалит все данные в локальной БД, и при следующем запуске она будет создана и заполнена тестовыми данными с нуля. Полезно при переходе на новую версию БД).*
